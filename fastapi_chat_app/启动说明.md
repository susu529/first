# 问题诊断和解决方法

## 问题原因

页面加载不出来的主要原因是：**FastAPI 和相关依赖包没有安装**。

## 解决方法

### 步骤 1：安装依赖包

在项目根目录（`fastapi_chat_app`）下运行：

```bash
pip install -r requirements.txt
```

或者如果 pip 不可用，尝试：

```bash
python -m pip install -r requirements.txt
```

### 步骤 2：启动服务

安装完成后，运行：

```bash
python run.py
```

或者：

```bash
cd fastapi_chat_app
python run.py
```

### 步骤 3：访问页面

服务启动后（会显示类似 "Uvicorn running on http://0.0.0.0:8000" 的信息），在浏览器中访问：

- **首页**：http://localhost:8000/
- **聊天页面**：http://localhost:8000/chat.html
- **文档管理**：http://localhost:8000/documents.html

## 其他可能的问题

### 问题 1：端口被占用

如果 8000 端口被占用，可以设置环境变量使用其他端口：

```bash
# Windows PowerShell
$env:PORT=8080
python run.py

# Windows CMD
set PORT=8080
python run.py
```

然后访问 http://localhost:8080/

### 问题 2：路径问题

如果仍然无法加载，检查：

1. `frontend` 文件夹是否存在
2. `frontend/index.html`、`frontend/chat.html`、`frontend/documents.html` 是否存在
3. 查看服务启动时的日志输出，确认路径信息

### 问题 3：浏览器控制台错误

打开浏览器开发者工具（F12），查看 Console 和 Network 标签页：

- 如果看到 404 错误，说明路由配置有问题
- 如果看到 CORS 错误，检查 CORS 配置
- 如果看到文件加载失败，检查静态文件路径

## 验证服务是否正常运行

在浏览器中访问：http://localhost:8000/docs

如果能看到 FastAPI 的自动生成的 API 文档页面，说明服务正常运行。

## 快速检查清单

- [ ] 已安装所有依赖包（`pip install -r requirements.txt`）
- [ ] 服务已启动（运行 `python run.py`）
- [ ] 8000 端口未被占用
- [ ] `frontend` 文件夹存在且包含所有文件
- [ ] 浏览器控制台没有错误
- [ ] 可以访问 http://localhost:8000/docs
